# 診断＋プレゼント教材 設計仕様（確定分）

最終更新：2026-07-21

現在LPに実装済みの診断は「暫定版（全15LP共通の3問・合格可能性のみ）」。
本書は、それを作り替えるための**確定した設計方針**。次回はここから詳細（文言）を詰めて実装する。

---

## 0. 全体の前提

- 導線：**LP内で診断 → 結果表示 → LINE登録でプレゼント → 体験URLへ**
- **プレゼント＝全LP「教材」に統一**（BOOK/図鑑案は不採用）
  - 理由：図鑑は比較検討モードに入らせ、LINE登録率は上がってもカウンセリング受講率が下がるため
- **教材＝ダウンロード物ではなく、URL遷移でのインタラクティブ学習体験**（実際のFAST-UPの学習環境を体感させる）

### 体験でできること（この4つのみ）
1. **毎日の計画**
2. **独自教材（ライブラリ上）**
3. **ミニ模試**（セクションごとの確認テストをWEB上でクイズ形式）
4. **学習管理**（学習時間・ミニ模試の理解度などを振り返れる）

> 注意：**週1固定面談・FAST-UP AI は体験では提供できない**ため、診断結果の接続先に使わない。

## 3. グループB 結果4タイプ（体験4機能と1:1で対応）

**設計原則**：4タイプすべてFAST-UPが提供し、かつ体験で必ず届けられる機能に接続する。他カテゴリの塾（映像授業・集団授業等）に誘導する選択肢は作らない。塾名は一切出さない。

| タイプ | どんな人 | 接続する体験機能 | オファー文（回収） |
|---|---|---|---|
| 毎日管理型 | 今日やることが決まらないと動けない | 毎日の計画 | 毎日やることが決まる状態を、そのまま体験できます |
| 教材特化型 | 何をどの順でやるかが定まらない | 独自教材ライブラリ | 全科目の教材と進める順番を、そのまま開放します |
| テスト反復型 | やったつもりで終わり、定着しない | ミニ模試 | セクションごとの確認テストで、定着を今日試せます |
| データ分析型 | 勉強しているのに伸びている実感がない | 学習管理 | 学習時間と理解度が可視化される画面を体験できます |

## 4. 結果の出力構造（全15LP共通・4ブロック）

```
① 結果       … 合格可能性%（A） or サポートタイプ（B）
② 詳細       … なぜその結果なのか（約100字）
③ 今後のアドバイス … 何をどの順でやるべきか（約100字）
④ オファー    … 「そのアドバイスを、今日から実行できる環境を用意しました」
               → LINE登録 → 体験URL（判定に対応する機能から開始）
```
③で示した「やるべきこと」を④で回収する流れにする。
